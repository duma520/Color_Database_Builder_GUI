## 工具概述

RGB颜色数据库管理工具是一款专为颜色数据管理设计的图形化应用程序，它可以帮助用户：

- 创建和维护一个结构化的RGB颜色数据库
- 批量导入/导出颜色数据（支持CSV和JSON格式）
- 可视化预览颜色
- 高效管理大量颜色数据

**简单理解**：就像一个"颜色图书馆"，可以存储、查找和管理各种颜色及其名称。

### 4.2 批量导入颜色

**适用场景**：当你有一大批颜色数据需要添加到数据库中时

**操作步骤**：
1. 点击"批量导入颜色"按钮
2. 选择CSV或JSON格式的文件
3. 选择导入模式：
   - **追加模式**：保留现有数据，只添加新颜色
   - **替换模式**：清空数据库后导入新数据

**文件格式示例**：

CSV格式：
```
R,G,B,颜色名称
255,0,0,纯红色
0,255,0,纯绿色
0,0,255,纯蓝色
```

JSON格式：
```json
[
  {"r": 255, "g": 0, "b": 0, "name": "纯红色"},
  {"r": 0, "g": 255, "b": 0, "name": "纯绿色"},
  {"r": 0, "g": 0, "b": 255, "name": "纯蓝色"}
]
```

### 4.3 批量导出颜色

**适用场景**：需要备份数据库或与其他工具共享颜色数据时

**操作步骤**：
1. 点击"批量导出颜色"按钮
2. 选择保存位置和格式（CSV或JSON）
3. 等待导出完成

### 4.4 添加单个颜色

**适用场景**：需要添加少量特定颜色时

**操作步骤**：
1. 点击"添加颜色"按钮
2. 在弹出的对话框中：
   - 输入颜色名称
   - 设置RGB值（0-255）
   - 或点击"从调色板选择"使用颜色选择器
3. 点击"确定"保存

### 4.5 清空数据库

**注意**：此操作不可逆，请谨慎使用！

**操作步骤**：
1. 点击"清空数据库"按钮
2. 确认操作
3. 等待清空完成

## 技术细节

### 数据库结构
```sql
CREATE TABLE colors (
  r SMALLINT NOT NULL CHECK(r >= 0 AND r <= 255),
  g SMALLINT NOT NULL CHECK(g >= 0 AND g <= 255),
  b SMALLINT NOT NULL CHECK(b >= 0 AND b <= 255),
  name TEXT NOT NULL,
  PRIMARY KEY (r, g, b)
)
```

## 常见问题解答

### Q: 导入大量数据时程序无响应？
A: 这是正常现象，程序正在后台处理数据。请查看进度条和日志了解当前状态。

### Q: 为什么有些颜色导入失败？
A: 可能原因：
- RGB值超出0-255范围
- 数据格式不正确
- 颜色已存在（在追加模式下不会覆盖）

### Q: 如何提高导入/导出速度？
A: 尝试调整"批量处理大小"参数（在"性能选项"中），通常较大的值会提高性能。
